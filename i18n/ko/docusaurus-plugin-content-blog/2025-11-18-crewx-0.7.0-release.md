CrewX 0.7.0이 출시되었습니다! 이번 릴리스의 핵심 기능은 **프로젝트 스캐폴딩(Project Scaffolding)**입니다. `crewx template` 명령어로 사전 구성된 AI 에이전트 설정과 프로젝트 문서를 손쉽게 설치하여 CrewX를 바로 활용할 수 있습니다.

<!--truncate-->

## 🎯 주요 신기능

### 프로젝트 스캐폴딩 (Project Scaffolding)

CrewX 0.7.0은 **프로젝트 템플릿 설치 기능**을 추가했습니다. `crewx template` 명령어로 AI 에이전트 설정과 프로젝트 문서를 한 번에 가져올 수 있어, 복잡한 설정 과정 없이 바로 CrewX를 사용할 수 있습니다.

```bash
# 템플릿 목록 보기
crewx template list

# 템플릿 설치
crewx template init wbs-automation
```

앞으로 더 다양한 템플릿이 추가될 예정입니다.

**커스텀 템플릿 저장소 사용:**

환경변수로 본인만의 템플릿 저장소를 지정할 수도 있습니다:

```bash
export CREWX_TEMPLATE_REPO=https://github.com/your-org/your-templates
crewx template list  # 본인 저장소의 템플릿 목록
crewx template init my-custom-template
```

### 첫 번째 템플릿: wbs-automation

현재 제공되는 **wbs-automation** 템플릿은 WBS(Work Breakdown Structure) 기반 자동화된 프로젝트 관리를 쉽게 시작할 수 있도록 구성되어 있습니다.

**핵심 구성:**
- `@coordinator` - WBS 기반 작업 조율 에이전트
- `wbs-loop.sh` - 30분 주기로 coordinator를 실행하는 자동화 스크립트
- `wbs.md` - 작업 분해 구조(WBS) 문서

**동작 방식:**
1. `wbs-loop.sh`가 30분 주기로 `@coordinator` 에이전트를 호출
2. Coordinator가 `wbs.md`를 분석하여 미완료 작업 파악
3. 작업 복잡도에 따라 `@claude`, `@codex` 등 CrewX 내장 에이전트들에게 병렬 배분
4. 각 작업 완료 후 `wbs.md`에 진행 상황 자동 업데이트

## 🚀 빠른 시작

### Step 1: CrewX 0.7.0 설치

```bash
npm install -g crewx@latest
crewx --version  # 0.7.0 확인
```

### Step 2: 템플릿 설치

```bash
# 템플릿 설치
crewx template init wbs-automation

# 설치된 에이전트 확인
crewx agent ls
```

### Step 3: 자동화 실행

```bash
# wbs.md 파일 작성 (프로젝트 작업 정의)
# 그리고 자동화 스크립트 실행
./wbs-loop.sh

# 또는 수동으로 coordinator 호출
crewx x "@coordinator wbs.md를 확인하고 작업을 진행해줘"
```
