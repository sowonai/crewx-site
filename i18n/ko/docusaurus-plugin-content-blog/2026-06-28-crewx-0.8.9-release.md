CrewX 0.8.9는 UI 중심 기능 7가지를 새롭게 선보입니다. 드래그앤드롭 상태 관리가 가능한 작업지시 보드, Codex/Claude/GLM 구독 한도를 한눈에 확인하는 사이드바 위젯, 터미널 없이 Git 변경 사항 탐색 및 저장, 문서 하이라이트와 주석, 그리고 메모리 압축 조절 기능이 추가됩니다.

<!--truncate-->

## 🎯 주요 기능

### 작업지시 보드 (WIUI001)

`docs/wi` 작업지시를 UI 안에서 바로 탐색하고 관리할 수 있습니다.

- 상태별로 정리된 작업지시 **보드/목록 뷰**
- **카드 드래그**로 상태 변경 (예: 할 일 → 진행 중 → 완료)
- **사이드 드로어**에서 WI 원문 Markdown을 바로 읽기

WI 파일을 직접 편집할 필요 없이, 보드에서 무엇이 할당되고 진행 중이며 완료됐는지 한눈에 파악할 수 있습니다.

### 구독 한도 위젯 (USGUI003)

사이드바 위젯이 구독 할당량을 항상 눈에 띄는 위치에 표시합니다.

- **Codex, Claude, GLM** 잔여 할당량을 사이드바 컴팩트 칩으로 상시 표시
- **클릭하면 팝오버**로 확장되어 5시간/주간 한도와 리셋 타이머 확인 가능
- 설정에서 **Provider별 표시 토글**로 원하는 프로바이더만 표시

별도 설정 페이지로 이동하지 않고도 사용량을 실시간으로 파악할 수 있습니다.

### Git 저장 시점 UI

이제 터미널을 열지 않고도 워크스페이스의 Git 변경 사항을 확인하고 저장 시점을 생성할 수 있습니다.

- 디렉토리별로 정리된 변경 파일 **트리 뷰**
- 스테이징할 파일을 정확히 선택하는 **디렉토리별 체크박스**
- 커밋 전 변경 내용을 검토하는 **나란히 보기(diff)** 화면
- 메시지와 함께 저장 시점(커밋)을 패널에서 바로 생성

터미널을 전환하지 않고도 언제든 에이전트 작업을 체크포인트로 남길 수 있습니다.

### 문서 하이라이트 & 주석 시스템

문서 뷰어에서 인라인 하이라이트와 코멘트 콜아웃을 지원합니다.

- **노란색, 초록색** 등 다양한 색상으로 텍스트 하이라이트
- 강조 구절에 앵커된 **코멘트 콜아웃** 추가
- 하이라이트는 **자동으로 중복 제거**되며, 개별 삭제도 가능
- 주석이 달린 문서는 **OKF 메타데이터**를 얻어 가중 검색에서 더 높은 순위로 노출

에이전트가 생성한 문서를 검토하고 인라인 피드백을 남기는 데 유용합니다.

### 메모리 감소율 (`decayRate`)

장시간 실행되는 에이전트는 활동 로그가 쌓여 컨텍스트 창을 압박할 수 있습니다. 새로운 `decayRate` 설정으로 드리밍 연속성 시스템이 오래된 로그 항목을 압축하는 속도를 조절할 수 있습니다.

- 설정 UI에서 **태스크별**로 구성 가능
- 낮은 값은 더 많은 이력을 보존하고, 높은 값은 더 빠르게 압축
- 최신 컨텍스트를 버리지 않으면서 토큰 비대화 방지

### 활동 로그 항목 크기 제한 (`maxEntryChars`)

각 활동 로그 항목에 최대 문자 수 상한을 설정할 수 있습니다. 상한을 초과한 항목은 마커와 함께 잘려, 도구 출력이 많은 태스크에서도 컨텍스트 창 크기를 예측 가능하게 유지합니다.

UI의 **설정 카드**에서 직접 상한값을 조정하세요.
